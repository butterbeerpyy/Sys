## 一、 课题定位与后续规划

本课题的核心目标并非在现阶段盲目追求实现支持大尺寸视觉语言模型（VLM）的完整复杂芯片，而是聚焦于大模型底层最具共性、计算量占比最高的**核心算子——矩阵乘法（Matrix Multiplication）进行硬件加速研究与原型验证**。

* **立项依据**：无论是 Transformer、CNN 还是各类现代视觉语言模型，其底层算力瓶颈均源于密集的矩阵乘加运算。若初期直接构建完整的计算系统，易因规模过大而导致核心体系结构设计无法做精做透。
* **阶段路线**：本阶段首先构建了一个“通用轻量级控制核 + 专用矩阵运算外设”的SoC原型系统，旨在率先打通控制流与计算流的分离调度、数据协同通路以及总线互联等底层逻辑。
* **老师建议**：在现有 2x2 矩阵加速原型稳定下来的基础上，后续两个方向都继续往下做，不做二选一。
  1. **脉动阵列方向**：从单个 2x2 计算块向更规则的数据流阵列扩展，重点提升并行度、吞吐量和数据复用效率。
  2. **自定义指令方向**：在现有 memory-mapped 控制方式之外，进一步探索更贴近 CPU 的指令级控制接口，提高软硬件协同效率。
* **后续节奏**：先把当前原型的功能、接口和仿真验证做扎实，再分别推进这两条线的最小可行版本，后续根据实现效果再比较哪条路线更适合继续深入。

## 二、 后续推进思路与阶段目标

老师已经明确建议，初期汇报里提到的两个方向都继续往下做。因此，后续工作不再按“只能选一个方向”来推进，而是以当前 2x2 矩阵加速原型为基础，分别向两个方向做并行扩展：

* **方向 A：脉动阵列 / 数据流阵列化**
  - 目标是把当前单个矩阵块的计算方式，进一步整理成更适合大规模并行计算的数据流结构。
  - 重点关注数据复用、并行计算和阵列扩展方式。
  - 这个方向更偏向体系结构和计算单元组织。

* **方向 B：自定义指令 / 更紧密的 CPU 接口**
  - 目标是在现有 memory-mapped 控制方式之外，探索更高效的 CPU 控制接口。
  - 重点关注 CPU 如何更自然地触发矩阵计算、减少控制开销，并提升软硬件协同效率。
  - 这个方向更偏向指令级接口和系统集成。

* **阶段性安排**
  - 先保证当前 2x2 原型的功能闭环、接口稳定和仿真正确。
  - 再分别实现两个方向的最小可行版本，避免一开始就把系统做得过大。
  - 最后结合实现复杂度、验证成本和课题展示效果，再决定后续重点深化的路线。
