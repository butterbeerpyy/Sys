# CPU vs DMA 性能对比测试

## 概述

本测试对比了 **CPU 直写模式** 和 **DMA 模式** 在两种硬件加速器上的性能表现：
- **VMAC**: 矩阵乘法加速器（2D 和 3D batch）
- **VLM**: 图像预处理加速器（112×112 像素）

## 测试场景

| 测试 ID | 工作负载 | 描述 |
|---------|----------|------|
| 1 | VMAC_2D_4x4 | 4×4 矩阵乘法 |
| 2 | VMAC_2D_8x8 | 8×8 矩阵乘法 |
| 3 | VMAC_3D_batch2_4x4 | 2 批次 4×4 矩阵乘法 |
| 4 | VMAC_3D_batch4_8x8 | 4 批次 8×8 矩阵乘法 |
| 5 | VLM_112x112 | 112×112 图像预处理 (12544 像素) |

## 关键结果

### 执行周期对比

| 工作负载 | CPU 模式 | DMA 模式 | 加速比 |
|----------|----------|----------|--------|
| VMAC 2D 4×4 | 450 | 180 | **2.50×** |
| VMAC 2D 8×8 | 850 | 271 | **3.14×** |
| VMAC 3D batch=2 4×4 | 920 | 350 | **2.63×** |
| VMAC 3D batch=4 8×8 | 3,600 | 950 | **3.79×** |
| VLM 112×112 | 25,100 | 12,615 | **1.99×** |

### 统计摘要

- **平均加速比**: 2.81×
- **最大加速比**: 3.79× (VMAC 3D batch=4)
- **最小加速比**: 1.99× (VLM)

## 性能分析

### VMAC（矩阵乘法加速器）

**CPU 直写模式瓶颈**：
- CPU 需要逐个写入矩阵元素（64 次写操作 for 8×8）
- 每次写操作需要地址译码、总线仲裁
- 3D batch 模式需要重复配置和数据传输

**DMA 模式优势**：
- 一次配置，硬件自动搬运矩阵数据
- Batch 模式可连续处理多个矩阵，无需 CPU 干预
- 更大的矩阵/batch 规模，加速比越明显

**观察**: 加速比随工作负载增大而提升（4×4: 2.50× → 8×8 batch=4: 3.79×）

### VLM（图像预处理加速器）

**CPU 直写模式瓶颈**：
- CPU 需要写 12,544 次（每像素一次）
- 即使单次写操作很快，累积开销巨大
- CPU 被长时间占用，无法处理其他任务

**DMA 模式优势**：
- 配置源地址后，DMA 自动读取全部像素
- CPU 只需 2 次配置操作（源地址 + 启动）
- ~12,600 周期完成，相比 CPU 模式节省 50% 时间

**观察**: VLM 的加速比（1.99×）低于 VMAC，因为：
1. 数据量更大（12,544 bytes vs 256 bytes）
2. DMA 传输本身需要时间（每周期 1 字节）
3. 但 CPU 解放出来的收益远大于绝对时间差异

## 吞吐量对比

### VMAC
- **CPU 模式**: ~0.14 bytes/cycle (8×8)
- **DMA 模式**: ~0.47 bytes/cycle (8×8)
- **提升**: 3.4×

### VLM
- **CPU 模式**: ~0.50 bytes/cycle
- **DMA 模式**: ~0.99 bytes/cycle
- **提升**: 2.0×

## 结论

1. **DMA 模式在所有场景下均显著优于 CPU 直写**，加速比 2-4 倍
2. **数据密集型任务收益最大**: VLM 节省 12,485 周期（绝对值）
3. **Batch 操作特别适合 DMA**: 3D batch 加速比最高（3.79×）
4. **CPU 利用率大幅提升**: DMA 工作时 CPU 可处理其他任务

- **时钟周期**: 10ns (100 MHz)
- **内存**: 512 words (2KB)

- **VLM 图像**: 112×112 = 12,544 pixels

## 引用建议

> 实验表明，在矩阵乘法和图像预处理两类计算密集型任务中，采用 DMA 传输相比 CPU 直写可获得 2-4 倍的性能提升。对于 8×8 矩阵的 4 批次 3D 操作，DMA 模式仅需 950 周期，而 CPU 模式需要 3,600 周期，加速比达 3.79×。图像预处理任务（12,544 像素）的加速比为 1.99×，但绝对时间节省达 12,485 周期，CPU 利用率提升显著。
