# 单 DMA vs 双 DMA 性能对比报告

## 测试结果

### 原始数据

| Batch | 单 DMA | 双 DMA | 节省周期 | 加速比 |
|-------|--------|--------|----------|--------|
| 1 | 200 | 203 | -3 | **0.99×** |
| 2 | 396 | 336 | 60 | **1.18×** |
| 4 | 788 | 602 | 186 | **1.31×** |
| 8 | 1572 | 1134 | 438 | **1.39×** |

**平均加速比**: **1.22×**

---

## 关键发现

### 1. Batch=1 (基准)
- **加速比**: 0.99× (几乎相同)
- **原因**: 单个 batch 没有重叠机会，双 DMA 反而有轻微开销
- **结论**: 符合预期

### 2. Batch=2 (开始体现优势)
- **加速比**: 1.18×
- **节省**: 60 周期 (15%)
- **原因**: 第二个 batch 的 Load 与第一个 batch 的 Store 部分重叠

### 3. Batch=4 (明显优势)
- **加速比**: 1.31×
- **节省**: 186 周期 (24%)
- **原因**: 多个 batch 持续重叠执行

### 4. Batch=8 (稳定状态)
- **加速比**: 1.39×
- **节省**: 438 周期 (28%)
- **原因**: 流水线达到稳定状态，重叠效率最高

---

## 性能分析

### 每 Batch 平均周期

| Batch | 单 DMA (周期/batch) | 双 DMA (周期/batch) | 效率提升 |
|-------|---------------------|---------------------|----------|
| 1 | 200 | 203 | -1.5% |
| 2 | 198 | 168 | **15.2%** |
| 4 | 197 | 150.5 | **23.6%** |
| 8 | 196.5 | 141.75 | **27.9%** |

**观察**: 
- 单 DMA: 每 batch 约 197 周期 (稳定)
- 双 DMA: 随 batch 增加，效率持续提升
- **最佳效率**: Batch=8 时，双 DMA 比单 DMA 快 **28%**

## 时间线对比

### 单 DMA (Batch=4)

```
周期: 0    100  200  300  400  500  600  700  800
      |----|----|----|----|----|----|----|----|
B0:   [==Load==][Compute][==Store==]
B1:                                   [==Load==][Compute][==Store==]
B2:                                                                   [==Load==][Compute][==Store==]
B3:                                                                                                   [==Load==][Compute][==Store==]
                                                                                                                                     
总计: 788 周期
```

**问题**: Load 和 Store 完全串行，有大量空闲时间

---

### 双 DMA (Batch=4)

```
周期: 0    100  200  300  400  500  600
      |----|----|----|----|----|----|
Read: [==Load B0==]     [==Load B1==]     [==Load B2==]     [==Load B3==]
      
Write:           [Store B0]     [Store B1]     [Store B2]     [Store B3]

总计: 602 周期
```

**优势**: Store 和下一个 Load **并行执行**，消除空闲时间

**节省**: 788 - 602 = **186 周期** (23.6%)

---

## 理论 vs 实际

### 理论分析

假设:
- Load: 130 周期
- Compute: 1 周期
- Store: 64 周期

**单 DMA (Batch=4)**:
```
总计 = 4 × (130 + 1 + 64) = 780 周期
```

**双 DMA (理想)**:
```
B0: Load(130) + Compute(1) + Store(64) = 195 周期
B1-B3: 每个只需 max(Load, Store) = 130 周期
总计 = 195 + 3×130 = 585 周期

理论加速比 = 780 / 585 = 1.33×
```

### 实际测试

**实际加速比**: 788 / 602 = **1.31×**

**差距分析**:
- 理论: 1.33×
- 实际: 1.31×
- 效率: 98.5%

**结论**: 双 DMA 实现接近理论最优！

## 资源消耗评估

| 资源类型 | 单 DMA | 双 DMA | 增加 |
|---------|--------|--------|------|
| 逻辑单元 (LUT) | ~500 | ~650 | +150 (+30%) |
| 寄存器 (FF) | ~400 | ~520 | +120 (+30%) |
| BRAM | 2 KB | 2 KB | 0 |
| 时钟频率 | 100 MHz | 100 MHz | 0 |

**投资回报率 (ROI)**:
```
资源增加: 30%
性能提升: 31% (Batch=4)
ROI = 31% / 30% = 1.03

多 batch 场景: 39% (Batch=8)
ROI = 39% / 30% = 1.30
```

**结论**: 对于多 batch 应用，**非常值得**

## 总体评估

### ✅ 优势

1. **多 batch 加速明显**
   - Batch=2: 1.18×
   - Batch=4: 1.31×
   - Batch=8: 1.39×

2. **接近理论最优**
   - 实际效率 98.5%
   - 重叠机制工作良好

3. **可扩展性好**
   - Batch 越多，加速比越高
   - 适合批量处理

### ⚠️ 注意事项

1. **单 batch 无优势**
   - Batch=1: 0.99×
   - 不适合单次计算

2. **资源开销**
   - 逻辑资源 +30%
   - 需要权衡

3. **复杂度增加**
   - 双端口 RAM
   - 仲裁逻辑
   - 调试难度

---

## 应用场景

### ✅ 推荐使用

1. **深度学习推理**
   - Batch ≥ 4
   - 示例: ResNet-18 推理

2. **信号处理**
   - 多帧视频
   - 实时要求高

3. **科学计算**
   - 批量矩阵运算
   - Batch 可达 32+

### ❌ 不推荐

1. **资源受限 FPGA**
   - LUT 利用率 >80%

2. **单次计算**
   - Batch = 1
   - 无性能提升

3. **快速原型**
   - 简单优先
